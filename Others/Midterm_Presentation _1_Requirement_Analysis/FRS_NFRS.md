
## 기능적 요구사항

vggt 파트

- co3Dv2 데이터셋을 JPEG, JPEG 2000, HEIC 세 가지 코덱으로 QF 10~100 범위의 무작위 압축률을 적용하여, 실험에 활용할 수 있는 데이터셋으로 준비해야 한다.
- VGGT 모델을 각 압축 버전 데이터셋에 대해 학습 및 평가해야 한다.
- VGGT 정량평가 지표는 기본적으로 Chamfer Distance를 사용한다.
- VGGT 평가 보완 지표로는 Relative Pose Accuracy (RPA), Depth metrics, 렌더드 뷰 품질(PSNR/SSIM/LPIPS) 등을 필요시 적용한다.
- 포인트 개수(Point Density), Object Detection 성능 등 Downstream task를 통해 추가 분석을 할 수 있다.
- 압축률, 성능지표와 모델 성능 간의 상관관계를 정량적으로 분석하고 시각화해야 한다.
- Input : Multi-view images / Output : 3D Point Cloud

3DGS 파트

- MipNeRF360 데이터셋 9개 씬 전체를 압축 후 디코딩(아래 두 방식 적용)하여 실험에 사용할 수 있도록 준비해야 한다.
1. HEVC: ffmpeg→YUV→HM Random Access 인코딩(QP: 27,32,37,42) → ffmpeg로 PNG 시퀀스 디코딩(또는 recon YUV 추출)
2. JPEG: 원본 이미지에 대해 QF = 10,30,50,70,90 적용
- 데이터셋 전처리 결과 주어진 이미지 시퀀스로 COLMAP을 실행해 카메라 포즈와 sparse 3D를 추정하여 3DGS 학습에 사용해야 한다.
- 3D Gaussian Splatting 모델을 사용해 압축 전후 성능을 비교해야 한다.
- 케이스 스터디 형식으로 압축률 변화에 따른 3DGS의 성능 변화 세 가지 평가지표(PSNR, SSIM, LPIPS)를 통해 분석하고 보고해야 한다.
- Input : PointCloud + Multi-view Images / Output : 3D Rendered Scene

**공통**

- 모든 실험 과정(압축 파라미터, ffmpeg/HM 명령어, COLMAP 설정, 3DGS 및 VGGT 모델 설정, 평가 결과)을 재현 가능하게 기록해야 한다.
- 필요 시 두 모델(VGGT, 3DGS)의 성능 개선 실험을 시도하고, 성능 향상 여부를 정량적으로 평가한다.

## 비기능적 요구사항

- **재현성** – 데이터 전처리, 압축, 학습, 평가 과정은 동일한 환경에서 반복 수행 가능해야 한다.
- **정확성** – 실험에 사용하는 데이터셋은 손상되지 않아야 하며, 압축/디코딩 과정에서 누락 프레임이 없어야 한다.
- **성능 측정 신뢰성** – 모든 성능 지표(PSNR, SSIM, LPIPS 등)는 통계적으로 유의미한 샘플 수(N ≥ 100 이상)를 기반으로 계산해야 하며, 결과는 평균값과 표준편차를 함께 제시해야 한다.
평가 기준 : A 랑 B..
- **확장성** – 추가 데이터셋이나 다른 코덱, 다른 모델에도 동일 파이프라인을 적용할 수 있어야 한다.
- **자동화** – 실험을 반복적으로 수행할 수 있도록 데이터 준비 및 평가 과정을 최대한 자동화한 스크립트를 제공야 한다.
- **시간 효율성** – 전체 실험이 프로젝트 기간 내 완료 가능해야 하며, 대규모 학습 시 GPU 리소스를 효율적으로 사용해야 한다.
- **문서화** – 실험 설정, 파라미터, 결과를 표준화된 형식으로 정리해 보고서/논문에 바로 활용 가능해야 한다.
- **비교 가능성** – 2D와 3D 파트에서 각각의 특성에 맞는 평가 지표를 사용하되, 결과를 동일한 형식(예: 표, 그래프 등)으로 정리하여 상호 비교·분석이 가능하도록 한다.
